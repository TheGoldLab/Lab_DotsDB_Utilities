dotsDB.dotsDB Module¶

The goal of this module is to provide a standard API to manage databases of random dots stimuli. The format for the databases is HDF5. The main functionalities are:

  1. create databases of random dots stimuli

    either by generating the stimuli or by providing existing stimuli

  2. append data to existing db

  3. inspect db (including parameters and other metadata about the stimuli)

  4. extract data from db

Logic¶

The idea is to generate the stimulus with a DotsStimulus instance and then write it to file with the write_stimulus_to_file() function.

A word on pixel frames and their HDF5 storage¶

Within this module, every pixel frame is handled as a 2D numpy array of booleans. The (i,j) entry in the array corresponds to the pixel at position (i,j) in space.

Note

For pixel with coordinates (i,j), small i corresponds to top, large i to bottom. Small y corresponds to left, large y to right.

Details on how such pixel frame is written to HDF5 file may be found in the documentation for the write_stimulus_to_file() function
